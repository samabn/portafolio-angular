PARA CREAR UN COMPONENTE DESDE LA LINEA DE COMANDOS ES CON EL COMANDO ng g c shared/header --spec=false LA OPCIÓN g ES PARA generate, LA c ES PARA component, shared ES LA 
CARPETA, SI ESTA NO EXISTE SERÁ CREADA, header ES EL NOMBRE DEL COMPONENTE Y --spec=false ES PARA QUE NO GENERE LOS ARCHIVOS PARA PRUEBAS UNITARIAS.

******************************************
EL ARCHIVO app-routing.module.ts ES EL ENCARGADO DE LA CREACION DE LAS RUTAS. USAR ESTE ARCHIVO PARA LAS RUTAS ES LA MANERA RECOMENDADA POR ANGULAR YA QUE ASÍ LA APLICACIÓN ES
MÁS FACIL DE MANTENER.

UN SERVICIO EN ANGULAR SE USA PARA COMPARTIR INFORMACIÓN GLOBAL A LO LARGO DE TODA LA APLICACIÓN, TAMBIEN SE USA PARA MANEJAR CIERTA LOGICA EN ALGÚN LUGAR ESPECIFICO DE LA PÁ
GINA.
PARA GENERAR UN SERVICIO SE USA EL COMANDO ng g s [ruta]/nombre_servicio skipTests.
LOS SERVICIOS SE COLOCAN EN LA PARTE DE LOS providers DEL ARCHIVO app.modules.ts, PERO ESTO NO BASTA, PARA PODER USARLO SE DEBE PONER EN EL ARCHIVO app.component.ts

--- CREAR VERSION DE PRODUCCIÓN ---
1. USAR EL COMANDO ng build --prod, ESTE CREARA UNA CARPETA LLAMADA dist ESTA TENDRÁ EN SU INTERIOR UNA CARPETA CON EL NOMBRE DEL PROYECTO, ESTA LA SE DEBE SUBIR AL
   SERVIDOR.
2. INSTALAR EL PAQUETE http-server DE NODE PARA HACER PRUEBAS LOCALES npm i -g http-server.
3. NAVEGAR A LA CARPETA dist/[nombre_proyecto] Y EJECUTAR EL COMANDO http-server -o PARA INICAR EL SERVIDOR.

--- DESPLEGAR LA APLICACIÓN EN GITHUB ---
1. CREAR UN NUEVO REPOSITORIIO EN GITHUB.
2. RENOMBRAR LA CARPETA DEL PROYECTO POR docs, PARA QUE GITHUB PUEDA DESPLEGAR LA APLICACION EN GITHUB PAGES.
3. EDITAR EL ARCHIVO index.html QUE ESTÁ EN LA CARPETA DEL PROYECTO QUE VAMOS A DESPLEGAR, BORRAR O COMENTAR LA LINEA <base href="/">, HACER LO MISMO CON EL ARCHIVO QUE ESTÁ
   EN LA CARPETA src/index.html. SE EDITAN ESTOS ARCHIVOS PORQUE SI NO LO HACEMOS TODAS LAS RUTAS VAN A APUNTAR AL ORIGEN DEL REPOSITORIO Y EN GITHUB ESTO NO ES ASÍ.
